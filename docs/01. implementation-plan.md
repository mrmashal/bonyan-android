## Project Overview

### Purpose
Bonyan is designed to:
- Manage family information in a hierarchical tree structure
- Track missions with components like responsibilities, items, budgets, notes, discussions, polls, and feedback
- Manage organizations with access levels and membership systems
- Implement a social reputation (manzelat) system based on feedback
- Provide offline-first functionality with server synchronization

### Key Features
1. **Messenger-Style Interface**: Each mission functions like a chat channel with notifications
2. **Bottom Navigation**: Notifications, Planner, Showcase (Vitrin), Profile
3. **Family Tree Management**: Visual representation of family relationships
4. **Mission Management**: Hierarchical missions with multiple components
5. **Organization System**: Membership levels and reputation tracking
6. **Offline Support**: Full functionality without internet connection

## Architecture & Design Principles

### Core Principles

1. **Reuse Telegram Components**: Maximize use of existing Telegram UI components (Cells, Dialogs, RecyclerLists, etc.)

2. **Offline-First Architecture**: All data stored locally in SQLite, synced with server when online

3. **Modular Design**: Clear separation between UI, business logic, and data layers

4. **Persian Calendar Support**: General calendar system support for all dates (e.g. Gregorian/Persian (Jalali) calendar)

5. **RTL Layout**: Full right-to-left layout support for Persian/Arabic text

### Technology Stack

| Layer | Technology |
|-------|------------|
| Language | Java |
| Database | SQLite (via Telegram wrapper) |
| UI Framework | Android SDK + Telegram UI Components |
| Networking | Telegram MTProto API |
| Threading | Telegram DispatchQueue |
| Image Loading | Telegram ImageLoader |

#### System 5: Reputation (Manzelat) System

**Files to Create:**
- `ReputationView.java`
- `FeedbackDialog.java`
- `LeaderboardActivity.java`

**Features:**
1. Reputation point calculation:
   - Base points from access level
   - Bonus from other organizations
   - Points from positive feedback
   - Penalty from negative feedback
2. Feedback system:
   - Like/positive feedback (+10 points typically)
   - Negative feedback (-2 for receiver, -1 for giver)
   - Feedback from higher access = more points
3. Anti-gaming measures:
   - Rate limiting on feedback
   - Detection of suspicious patterns
4. Leaderboards and rankings

**Reputation Calculation Formula:**
```
Base = AccessLevelMinPoints
Bonus = OtherOrgsBonus (if rep > 200 in another org = +100)
Feedback = Sum of all feedback points
Total = Base + Bonus + Feedback
```

**Implementation Steps:**
1. Create reputation calculation logic
2. Implement feedback system
3. Add anti-gaming measures
4. Create leaderboard views
5. Integrate with organization membership
